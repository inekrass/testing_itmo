# Задание: тестирование интернет-магазина книг

Вам выдан рабочий проект серверной части интернет-магазина книг. Тесты к нему не написаны — это ваша задача.

Перед началом изучите техническое задание в `docs/REQUIREMENTS.md`. Тесты пишутся по требованиям, а не только по коду: ваша цель — проверить, что система ведёт себя так, как описано в ТЗ.

## Что нужно сделать

Покрыть проект тестами разных видов. Распределяйте проверки по уровням осознанно — каждый вид тестов решает свою задачу.

### 1. Модульные тесты расчёта стоимости

Покрыть `PricingService` (`src/bookstore/pricing.py`)

### 2. Тесты с подменой зависимостей

Покрыть `OrderService.checkout` (`src/bookstore/order_service.py`)

### 3. Тесты HTTP API

Покрыть `api.py` через `TestClient` с подменой зависимостей (`app.dependency_overrides`):

- Проверка статус-кодов: 201, 402, 403, 404, 409, 422, 502.
- Проверка тела ответа на соответствие схеме.
- Проверка валидации входных данных (422 на некорректное тело).

### 4. Тесты на основе свойств (property-based)

Сформулировать и проверить инварианты `PricingService` с помощью Hypothesis

### 5. Мутационное тестирование

- Запустить `cosmic-ray` на модуле `pricing.py`.
- Разобрать выживших мутантов и при необходимости дополнить тесты.

## Что сдавать

1. Каталог `tests/` с тестами, разложенными по видам.
2. Краткий отчёт по мутационному тестированию.
3. Если найдёте расхождение между кодом и ТЗ — опишите его отдельно. Код при этом менять не нужно.
