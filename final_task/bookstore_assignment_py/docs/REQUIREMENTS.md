# Техническое задание

## Интернет-магазин книг

---

## 1. Назначение

Система обслуживает оформление и оплату заказов в интернет-магазине книг. Покупатель собирает заказ из книг каталога, применяет промокод, указывает адрес доставки и оплачивает заказ. Система рассчитывает итоговую стоимость с учётом скидок, доставки и налога, проверяет наличие книг на складе, проводит оплату и отправляет покупателю подтверждение.

Дополнительно система подбирает покупателю рекомендации книг.

Расчёты с внешними системами (склад, платёжный шлюз, уведомления, курсы валют) выполняются через отдельные сервисы, которые подключаются к приложению как зависимости.

---

## 2. Глоссарий

| Термин | Значение |
|--------|----------|
| Книга (Book) | Товар каталога с ISBN, ценой, весом и категорией |
| Категория (BookCategory) | Тип книги: художественная, нехудожественная, учебник, детская, коллекционная |
| Покупатель (Customer) | Клиент магазина. Имеет уровень, влияющий на скидку и лимит заказа |
| Уровень (CustomerTier) | Сегмент покупателя: BRONZE, SILVER, GOLD |
| Позиция (CartItem) | Книга и её количество внутри заказа |
| Заказ (Order) | Набор позиций одного покупателя с текущим статусом |
| Промокод | Код, дающий скидку на заказ |
| Доставка | Стоимость доставки заказа, зависит от суммы и веса |
| Резервация | Бронирование товара на складе под конкретный заказ |

---

## 3. Сущности

### 3.1. Книга (Book)

Неизменяемый объект каталога.

| Поле | Тип | Описание |
|------|-----|----------|
| isbn | строка | Уникальный идентификатор книги |
| title | строка | Название |
| author | строка | Автор |
| category | BookCategory | Категория |
| base_price | Decimal | Базовая цена в рублях |
| weight_grams | целое | Вес в граммах |
| publication_year | целое | Год издания |

Категории: `FICTION` (художественная), `NON_FICTION` (нехудожественная), `TEXTBOOK` (учебник), `CHILDREN` (детская), `RARE` (коллекционная).

### 3.2. Покупатель (Customer)

| Поле | Тип | Описание |
|------|-----|----------|
| customer_id | строка | Идентификатор |
| name | строка | Имя |
| email | строка | Электронная почта |
| tier | CustomerTier | Уровень: BRONZE, SILVER, GOLD |
| orders_last_year | целое | Число заказов за прошлый год |
| country | строка | Страна (по умолчанию RU) |
| is_blocked | булево | Заблокирован ли покупатель |

### 3.3. Позиция заказа (CartItem)

Книга и её количество. Стоимость позиции (`line_total`) равна базовой цене книги, умноженной на количество.

### 3.4. Заказ (Order)

| Поле | Тип | Описание |
|------|-----|----------|
| order_id | строка | Идентификатор |
| customer | Customer | Покупатель |
| items | список CartItem | Позиции заказа |
| status | OrderStatus | Текущий статус |
| promo_code | строка или пусто | Применённый промокод |
| shipping_address | Address или пусто | Адрес доставки |
| payment_transaction_id | строка или пусто | Идентификатор платежа |

Вычисляемые свойства: `subtotal` (сумма позиций по базовым ценам), `total_weight_grams` (суммарный вес), `is_empty` (пуст ли заказ).

Статусы заказа: `DRAFT` (черновик), `PENDING_PAYMENT` (ожидает оплаты), `PAID` (оплачен), `SHIPPED` (отправлен), `DELIVERED` (доставлен), `CANCELLED` (отменён), `REFUNDED` (возвращён).

---

## 4. Функциональные требования

### 4.1. Работа с заказом

- **Создание заказа.** Для покупателя создаётся пустой заказ в статусе `DRAFT`. Заблокированному покупателю заказ не создаётся.
- **Добавление позиции.** В черновик добавляется книга с указанным количеством. Если книга уже есть в заказе, её количество увеличивается. Количество должно быть положительным. Книга должна существовать в каталоге.
- **Удаление позиции.** Из черновика удаляется книга по ISBN.
- **Применение промокода.** К черновику привязывается промокод. Его корректность проверяется при расчёте стоимости.

Изменять позиции и промокод можно только у заказа в статусе `DRAFT`.

### 4.2. Расчёт стоимости

Система рассчитывает стоимость заказа с разбивкой: сумма позиций, размер скидки, сумма после скидки, доставка, налог и итог. Правила расчёта приведены в разделе 5.

### 4.3. Оформление и оплата (checkout)

Оформление выполняется по шагам:

1. Проверка заказа по бизнес-правилам (раздел 5.5).
2. Проверка наличия каждой позиции на складе.
3. Резервирование всех позиций на складе.
4. Расчёт итоговой стоимости.
5. Списание средств через платёжный шлюз.
6. Перевод заказа в статус `PAID`.
7. Отправка подтверждения покупателю.

Поведение при сбоях:

- Если товара на складе недостаточно — заказ не оформляется, статус не меняется.
- Если при резервировании произошёл сбой — ранее сделанные резервации снимаются.
- Если оплата отклонена или платёжный шлюз вернул ошибку — резервации снимаются, заказ переводится в статус `CANCELLED`.
- Если отправка подтверждения завершилась ошибкой — заказ всё равно считается оплаченным (статус `PAID`).

### 4.4. Отмена заказа

Заказ можно отменить из статусов `DRAFT`, `PENDING_PAYMENT` и `PAID`. Для оплаченного заказа выполняется возврат средств, и статус становится `REFUNDED`; в остальных случаях — `CANCELLED`. Отмена доставленного, уже отменённого или возвращённого заказа не допускается.

### 4.5. Рекомендации

Система подбирает покупателю до N рекомендаций книг через внешний ML-сервис. Если сервис недоступен, применяется запасной подбор по популярным книгам художественной категории. Заблокированному покупателю рекомендации не выдаются.

## 5. Бизнес-правила

### 5.1. Скидка по уровню покупателя

| Уровень | Скидка |
|---------|--------|
| BRONZE | 0% |
| SILVER | 5% |
| GOLD | 10% |

### 5.2. Промокоды

| Код | Скидка | Условие |
|-----|--------|---------|
| WELCOME15 | 15% | — |
| SUMMER20 | 20% | — |
| STUDENT25 | 25% | Только если все позиции заказа — учебники (TEXTBOOK) |

Неизвестный промокод считается ошибкой расчёта.

### 5.3. Выбор скидки

К заказу применяется **одна** скидка — наибольшая из подходящих: по уровню покупателя, по промокоду и по распродаже «чёрная пятница». Скидка начисляется только на стоимость книг, не относящихся к коллекционным.

Распродажа «чёрная пятница» действует с 24 по 30 ноября включительно и даёт скидку 30%.

Книги категории `RARE` (коллекционные) не участвуют в скидках: их стоимость прибавляется к итогу полностью.

### 5.4. Доставка и налог

Стоимость доставки:

- Бесплатно, если в заказе есть хотя бы одна детская книга (`CHILDREN`).
- Бесплатно, если сумма заказа после скидки не меньше 3000 рублей.
- Иначе базовая ставка 300 рублей плюс 1 рубль за каждые полные 100 граммов веса свыше 1 килограмма.

Налог: НДС 10% начисляется на сумму заказа после скидки (без учёта доставки).

Итоговая стоимость = сумма после скидки + доставка + НДС.

### 5.5. Ограничения заказа

- Заказ не может быть пустым.
- Заблокированный покупатель не может оформлять заказы.
- Количество одной книги в заказе — от 1 до 20.
- Сумма заказа не должна превышать лимит уровня: для GOLD — 500 000 рублей, для остальных — 100 000 рублей.

### 5.6. Примеры расчёта

- SILVER, 2 книги по 1500 руб, вес 1200 г, без промокода, обычный день: сумма 3000, скидка 5% = 150, сумма после скидки 2850. Доставка платная (2850 < 3000): 300 + 2 = 302. НДС 285. Итого 3437.
- GOLD, заказ на 5000 руб, без промокода: скидка 10% = 500, сумма после скидки 4500. Доставка бесплатна (≥ 3000). НДС 450. Итого 4950.
- BRONZE, промокод WELCOME15 на заказ 2000 руб: применяется скидка 15% (больше уровневой 0%), скидка 300, сумма после скидки 1700.
- Заказ только из учебников с промокодом STUDENT25: скидка 25%. Тот же промокод на заказ со смешанными категориями скидки не даёт.

## 6. Внешние зависимости

Система не содержит реализаций перечисленных сервисов — они подключаются к приложению как зависимости.

| Зависимость | Назначение |
|-------------|------------|
| BookRepository | Каталог книг: получение книги по ISBN, поиск |
| InventoryService | Склад: остаток, резервирование, снятие резервации |
| PaymentGateway | Платёжный шлюз: списание и возврат средств |
| NotificationService | Уведомления покупателю (подтверждение заказа, отправка) |
| CurrencyConverter | Конвертация валют по курсу |
| Clock | Источник текущего времени |
| MLRecommendationProvider | Внешний сервис рекомендаций |

## 7. HTTP API

Система предоставляет REST API. Ошибки бизнес-логики отображаются на статусы HTTP.

| Метод и путь | Назначение |
|--------------|------------|
| POST /orders | Создать заказ |
| GET /orders/{id} | Получить заказ |
| POST /orders/{id}/items | Добавить позицию |
| DELETE /orders/{id}/items/{isbn} | Удалить позицию |
| POST /orders/{id}/promo | Применить промокод |
| POST /orders/{id}/checkout | Оформить и оплатить заказ |
| POST /orders/{id}/cancel | Отменить заказ |

Соответствие ситуаций и статусов:

| Ситуация | Статус |
|----------|--------|
| Заказ создан | 201 Created |
| Запрос выполнен | 200 OK |
| Покупатель или заказ не найден | 404 Not Found |
| Книга не найдена при добавлении | 404 Not Found |
| Невалидное тело запроса | 422 Unprocessable Entity |
| Нарушение бизнес-правила при расчёте | 422 Unprocessable Entity |
| Недопустимый статус заказа для операции | 409 Conflict |
| Недостаточно товара на складе | 409 Conflict |
| Платёж отклонён | 402 Payment Required |
| Ошибка платёжного шлюза | 502 Bad Gateway |
| Создание заказа заблокированному покупателю | 403 Forbidden |

### Пример

```
POST /orders/{id}/checkout
{
  "shipping_address": {
    "country": "RU",
    "city": "Москва",
    "postal_code": "101000",
    "street": "Тверская, 1"
  }
}

201 Created
{
  "order_id": "...",
  "status": "PAID",
  "total": 3437.00,
  "transaction_id": "TX-12345"
}
```
