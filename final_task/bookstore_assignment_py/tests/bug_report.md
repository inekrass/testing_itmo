# Баг-репорт: расхождения кода и ТЗ

## 1. Checkout API возвращает 200 вместо 201

По ТЗ успешное оформление заказа через `POST /orders/{id}/checkout` должно
возвращать статус `201 Created`.

Сейчас этот эндпоинт возвращает `200 OK`, потому что в `src/bookstore/api.py`
для обработчика `checkout` не указан `status_code=201`.

## 2. Резервации снимаются дважды при отклонённом платеже

Если платёж отклонён, сервис должен снять резервации один раз и отменить заказ.

Сейчас резервации могут сниматься дважды. Сначала они снимаются перед выбрасыванием
`PaymentDeclinedError`, а потом это же исключение попадает в общий обработчик
`PaymentError`, где резервации снимаются повторно.

## 3. `PaymentStatus.GATEWAY_ERROR` как результат платежа не отличён от отказа

По ТЗ ошибка платёжного шлюза должна отображаться в API как `502 Bad Gateway`.

Сейчас, если платёжный шлюз вернёт результат со статусом
`PaymentStatus.GATEWAY_ERROR`, код обработает его как обычный отклонённый платёж.
В таком случае API может вернуть не `502`, а ошибку отклонённого платежа.
